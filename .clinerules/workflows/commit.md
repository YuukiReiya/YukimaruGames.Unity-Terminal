# コミットワークフロー

変更内容を分析して適切なコミットメッセージを生成し、コミットを実行する。

---

## Step 1: 変更内容の確認

以下のコマンドで現在の変更状態を確認する:

```
git status
git diff
```

変更ファイルの一覧と差分を把握する。

---

## Step 2: コミット範囲の確認

> **注意: このワークフローはActモードであっても実行前に必ずユーザーに確認すること。**

ユーザーに以下を確認する:

- すべての変更をコミットするか、特定のファイルのみか
- コミットの粒度（1機能1コミットの原則に従う）

複数の関心事が混在している場合は分割を提案する。

---

## Step 3: コミットメッセージの生成・確認

差分を分析して以下の形式でコミットメッセージを提案する:

```
[feat]   新機能の追加
[fix]    バグ修正
[refactor] リファクタリング
[test]   テストの追加・修正
[docs]   ドキュメントの更新
[chore]  設定・環境変更
```

例:
```
[feat] PlayerHpにダメージ計算ロジックを追加
```

提案内容をユーザーに確認し、承認を得てから次のステップに進む。

---

## Step 4: コミットの実行

**重要：必ず `08-github-instruction.md` の「アイデンティティ確認フロー」に従うこと。**

承認されたメッセージでコミットを実行する:

1.  `.agents/git-credentials.txt` を読み込み、Agent情報を取得（なければユーザーに確認）。
2.  **ローカルコミットのみ**を実行する（`push` は含めないこと）：

```
git add [対象ファイル]
git commit -m "[種別] メッセージ"
```

実行結果を報告する。
