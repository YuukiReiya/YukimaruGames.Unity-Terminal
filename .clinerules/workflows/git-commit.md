# Workflow: git-commit

## 目的
ユーザー指定の範囲（包含・除外）に基づき、適切なプレフィックスを付けてコミットを完了させます。

## 手順
1. **ターゲット解析とステージング**:
   ユーザーの入力（例: `/commit *.cs !*.env`）から対象を特定し、以下の順にコマンドを実行する。
   - **ルート確認**: `ls .agents/git-credentials.txt` を実行し、ファイルが存在するディレクトリ（リポジトリルート）にいることを必ず確認する。
   - **リセット**: `git reset` で現在のステージングを完全に解除する。
   - **包含（!なし）**: 引数がある場合は `git --no-pager add <引数>...` を実行。ない場合は `git --no-pager add -A` を実行。
   - **除外（!あり）**: `!` 付きの引数がある場合、そのパスに対して `git --no-pager reset <引数（!を除く）>...` を実行する。

2. **分析対象の確定**:
   `git --no-pager status --porcelain` を実行。**「ステージング済み（左端がAまたはM）」のファイルのみ**を分析対象とする。除外したファイルの情報はメッセージに含めない。

3. **情報の取得**:
   改めて `.agents/git-credentials.txt` を読み取る。もしここで「見つからない」と判断しそうになったら、直ちに `pwd` を実行して現在の場所をユーザーに報告し、勝手な推論を中止すること。

4. **メッセージの構成**:
   - 手順2で確定した差分に基づき、適切なプレフィックスとメッセージを構成する。
   - **重要**: メッセージの内容についてユーザーに確認や質問をしてはいけません。プロジェクトの文脈と差分から、自信を持って最適なものを決定してください。
   - 形式は `[prefix] subject` とし、英語または日本語で構成します。

5. **Skillの呼び出し**:
   `SKILL.md` の `git-commit` スキルを使用してコミットを実行する。

6. **完了報告と待機**:
   実行結果を日本語で報告し、**ユーザーの指示があるまで Push 等の追加操作は行わず停止する。**
