# PRレビュー返信ワークフロー

PRのレビューコメントを取得し、内容を分析して返信・修正対応を行う。

---

## Step 1: 対象PRの確認

ユーザーにPR番号を確認する。
指定がない場合は現在のブランチのPRを自動で取得する:

```
gh pr list --state open
gh pr view
```

---

## Step 2: レビューコメントの取得

対象PRのレビューコメントをすべて取得する:

```
gh pr view [PR番号] --comments
```

以下の情報を整理して報告する:

- レビュアー名
- コメント内容
- 対象ファイル・行番号（あれば）
- Approve / Request Changes の状態

---

## Step 3: 対応方針の提案

各レビューコメントに対して以下のいずれかを提案し、ユーザーの承認を得る:

**A. コード修正が必要な場合**
```
【対応方針】修正する
【修正内容】〇〇を△△に変更する
【理由】〇〇のため
```

**B. 質問・説明が必要な場合**
```
【対応方針】返信で説明する
【返信内容】〇〇という意図で実装しました。△△のため現状の実装が適切と考えます。
```

**C. 指摘に同意して修正する場合**
```
【対応方針】修正する + 返信する
【返信内容】ご指摘ありがとうございます。〇〇に修正しました。
```

すべてのコメントへの対応方針をユーザーが確認するまで次のステップに進まない。

---

## Step 4: コード修正の実施

修正が必要なコメントに対してコードを修正する。
修正後は `/code-review.md` の観点で自己チェックする。

---

## Step 5: レビューへの返信投稿

承認された返信内容をGitHub APIで投稿する:

```
gh api \
  repos/{owner}/{repo}/pulls/{PR番号}/comments \
  --method POST \
  --field body="返信内容" \
  --field in_reply_to=[コメントID]
```

スレッドへの返信ではなく、PRへの全体コメントとして投稿する場合:

```
gh pr comment [PR番号] --body "返信内容"
```

---

## Step 6: 修正のコミット・プッシュ

コード修正がある場合は `/commit.md` を実行してコミットする。
その後プッシュする:

```
git push
```

完了後にPRのURLと対応内容のサマリーを報告する。
