# Workflow: git-commit

## 目的
変更内容を分析し、適切なプレフィックスを付けてコミットを完了させます。

## 手順
1. **差分分析**: `git --no-pager status --porcelain` および `git --no-pager diff --cached` を実行し、変更内容を把握する。
   - **注意**: 標示量が多い場合でもハングを避けるため、必ず`--no-pager`オプションを付与すること。
2. **情報の取得**: `.agents/git-credentials.txt` からコミット作成者情報を取得する。
   - **重要**: ls .agents/git-credentials.txt で存在を確認してから cat してください。もし見つからないと判断した場合は、一度 pwd で現在のディレクトリを確認してから再探査してください。
   - **ファイルがない場合**: ユーザーに日本語で報告し、設定を促す（あるいは設定用フローへ）。
3. **コミット内容の決定**: 分析結果に基づき、適切なプレフィックス（feat, add, fix, refactor 等）とメッセージを構成する。
4. **Skillの呼び出し**: `SKILL.md` に定義された `git-commit` スキルを使用してコミットを実行する。
5. **完了報告と待機**: 実行結果を日本語で報告し、**ユーザーの次の指示があるまで一切の追加コマンド（git push等）を実行せず停止してください。**
