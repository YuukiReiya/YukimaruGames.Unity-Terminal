# Workflow: git-commit

## 目的
変更内容を分析し、適切なプレフィックスを付けてコミットを完了させます。

## 手順
1. **差分分析**: `git status` および `git diff --cached` を実行し、変更内容を把握する。
2. **情報の取得**: `.agents/git-credentials.txt` からコミット作成者情報を取得する。
   - **ファイルがない場合**: ユーザーに日本語で報告し、設定を促す（あるいは設定用フローへ）。
3. **コミット内容の決定**: 分析結果に基づき、適切なプレフィックス（feat, add, fix, refactor 等）とメッセージを構成する。
4. **Skillの呼び出し**: `SKILL.md` に定義された `git-commit` スキルを使用してコミットを実行する。
5. **完了報告**: 実行結果を**日本語**でユーザーに報告する。
